# Release 1.0.1 and pull request design

## Goal

Prepare the completed **List all missing daily notes** change as version `1.0.1` and publish it for review in a draft pull request against `master`.

## Version synchronization

- Set `package.json` and `package-lock.json` to version `1.0.1`.
- Set `manifest.json` to version `1.0.1`.
- Add `"1.0.1": "1.13.0"` to `versions.json`, preserving the existing `1.0.0` mapping.
- Do not create a Git tag. Release tags belong after the pull request is merged and the release is ready to publish.

The files will be edited explicitly so the resulting diff is transparent and does not invoke npm's automatic commit/tag behavior.

## Verification

Run the complete unit test suite, ESLint, the production TypeScript/esbuild build, and `git diff --check`. Confirm all four version-bearing files agree on `1.0.1` and that `versions.json` maps it to the unchanged minimum Obsidian version `1.13.0`.

## Publication

Commit only the intended version files with the message `chore: release 1.0.1`. Push the existing `codex/list-missing-daily-notes` branch to `origin`, then open a draft pull request targeting `master`.

The pull request description will summarize the new list command, removal of the dry-run setting, synchronized version bump, user impact, and verification results.

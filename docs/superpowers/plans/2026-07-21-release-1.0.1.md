# Release 1.0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Synchronize the plugin version at `1.0.1`, verify the release candidate, and open a draft pull request against `master`.

**Architecture:** Update each version-bearing JSON file explicitly without invoking npm's automatic commit/tag flow. Keep version preparation and GitHub publication as separate tasks so the release candidate is committed and verified before it is pushed.

**Tech Stack:** npm package metadata, Obsidian plugin manifest/version map, Git, GitHub CLI or connected GitHub app.

## Global Constraints

- The target version is exactly `1.0.1`.
- Keep `minAppVersion` exactly `1.13.0`.
- Preserve the existing `"1.0.0": "1.13.0"` entry in `versions.json`.
- Do not create a Git tag.
- Commit only version-bearing files in the release commit with message `chore: release 1.0.1`.
- Push the existing `codex/list-missing-daily-notes` branch and open a draft pull request targeting `master`.

---

### Task 1: Synchronize and verify version 1.0.1

**Files:**
- Modify: `package.json:3`
- Modify: `package-lock.json:3,9`
- Modify: `manifest.json:4`
- Modify: `versions.json:2-3`

**Interfaces:**
- Consumes: current version `1.0.0` and minimum Obsidian version `1.13.0`.
- Produces: four synchronized release metadata files representing version `1.0.1`.

- [ ] **Step 1: Update package metadata**

Set the top-level version in `package.json` to:

```json
"version": "1.0.1"
```

Set both root-package version fields in `package-lock.json` to `1.0.1`, leaving dependency versions untouched.

- [ ] **Step 2: Update Obsidian release metadata**

Set `manifest.json` to:

```json
"version": "1.0.1",
"minAppVersion": "1.13.0"
```

Set `versions.json` to:

```json
{
	"1.0.0": "1.13.0",
	"1.0.1": "1.13.0"
}
```

- [ ] **Step 3: Verify synchronized metadata**

Run:

```bash
node -e "const fs=require('fs'); const p=JSON.parse(fs.readFileSync('package.json')); const l=JSON.parse(fs.readFileSync('package-lock.json')); const m=JSON.parse(fs.readFileSync('manifest.json')); const v=JSON.parse(fs.readFileSync('versions.json')); if (p.version !== '1.0.1' || l.version !== '1.0.1' || l.packages[''].version !== '1.0.1' || m.version !== '1.0.1' || m.minAppVersion !== '1.13.0' || v['1.0.0'] !== '1.13.0' || v['1.0.1'] !== '1.13.0') process.exit(1)"
```

Expected: exit 0 with no output.

- [ ] **Step 4: Run complete release verification**

Run `npm test`, `npm run lint`, `npm run build`, and `git diff --check`.

Expected: 7 tests pass; ESLint, TypeScript/esbuild, and whitespace checks exit 0.

- [ ] **Step 5: Commit only release metadata**

```bash
git add package.json package-lock.json manifest.json versions.json
git commit -m "chore: release 1.0.1"
```

Confirm `git status --short` is empty and no tag was created.
